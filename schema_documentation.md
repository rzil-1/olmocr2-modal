# OCR Output JSON Schema Documentation

This document outlines the JSON schema generated by the `modal_diagram_extractor.py` serverless OCR engine. It is designed to be easily ingested by your downstream Evaluation Engine.

## The JSON Structure

The output is a dictionary where the **keys** are the filenames of the processed PDFs, and the **values** are objects containing the extracted data.

```json
{
    "example_document.pdf": {
        "raw_markdown": "# Title\n\nThis is the fully extracted continuous text of the document...",
        "equations": [
            "\\frac{dD(t)}{dt}",
            "D(t) = 50 \\sin(t) \\implies \\frac{d(D(t))}{dt} = 50 \\cos(t)"
        ],
        "diagrams": [
            {
                "alt_text": "Graph showing demand and resource allocation over time",
                "page": 2,
                "coordinates": [320, 540, 1040, 480],
                "image_path": "detected_diagrams/example_document_pg2_0_Graph_showing.png"
            }
        ]
    }
}
```

### Field Definitions:
- **`raw_markdown` (String):** The complete, concatenated text of the entire PDF, formatted in clean Markdown. It includes headers, bullet points, and inline math markers.
- **`equations` (Array of Strings):** A clean array containing every single LaTeX equation (both inline `\(\)` and block `\[\]`) found in the document. This allows your Evaluation Engine to instantly iterate over the mathematical formulas without needing to parse the raw text itself.
- **`diagrams` (Array of Objects):** Contains metadata for every cropped image found in the PDF.
  - `alt_text`: The AI's semantic description of the diagram.
  - `page`: The page number (1-indexed) where the diagram was found.
  - `coordinates`: The `[x, y, width, height]` scaled coordinate box.
  - `image_path`: The local path to the cropped `.png` image that was saved. *(Note: If integrating via the direct Modal python API, the image bytes are returned in memory rather than saved to disk).*

## How to Customize the Schema for the Evaluation Engine

If your Evaluation Engine requires a different JSON structure (for example, you want pages separated instead of concatenated, or you want to add confidence scores), you can easily modify the schema inside **`modal_diagram_extractor.py`**.

### 1. Changing the Base Schema
Open `modal_diagram_extractor.py`. Around **Line 83**, you will find the initial dictionary definition. Change this to match your needed schema:

```python
# modal_diagram_extractor.py (Line 83)
results = {
    "raw_markdown": "",
    "equations": [],
    "diagrams": []
}
# Example Modification: Adding a page-by-page array
# results = {"pages": [], "metadata": {"status": "success"}}
```

### 2. Changing How Data is Appended
If you change the base schema, you must also update how the extracted data is pushed into it. Scroll down to the `for page_num in range(...)` loop.

Around **Line 128**, you will see how text is accumulated:
```python
# Text accumulation (Line 128)
results["raw_markdown"] += text_output + "\n\n"
```

Around **Line 131**, you will see how equations are pushed:
```python
# Equation extraction (Line 131)
inline_math = re.findall(r'\\\((.*?)\\\)', text_output, re.DOTALL)
results["equations"].extend([eq.strip() for eq in inline_math])
```

Around **Line 167**, you will see how diagram metadata is pushed:
```python
# Diagram appending (Line 167)
results["diagrams"].append({
    "alt_text": alt_text,
    "page": page_num + 1,
    "coordinates": [int(x), int(y), int(w), int(h)],
    "image_path": f"detected_diagrams/{clean_filename}"
})
```

**To summarize:** You are fully encouraged to change the `results` dictionary and how data is appended to it. However, do not change the Regex `pattern` itself or the PyMuPDF (`fitz`) cropping logic, as those are required to correctly parse the AI's raw output.
